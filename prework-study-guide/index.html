HTML

  • The head element contains information, or metadata, that helps the browser to render the page correctly.
  • The body element contains the content that is shown to the user.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage.
  • The style element defines the CSS styles associated with a website.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • Search engine optimization (SEO) is the process of improving the ranking of a website in search engines.
  • Heading elements are from h1 to h6, must be displayed in that order.

CSS

  • Inline CSS = Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Internal CSS = using a style element in the head element, internal style sheet contains CSS rules for the webpage in the head section of the HTML file.The rules only apply to that page.
  • External CSS = the most common way to style a webpage. it allows developers to keep all their CSS rules in a separate file, which makes design changes easier. When using an external style sheet, we link the file using an link element placed in the HTML's head.
  • The CSS link tag in detail:
    • link = creates the connection to an external file in general
    • rel = specifies the relationship between the current document and the linked document resource
    • href = tells the location of the external file
  • CSS Rule consists of:
    • The selector
    • The declaration
    • The property
    • The value
  • Margin = The space outside the element, after the border
  • Border = the line around the element, after the padding
  • Padding = The space around the element itself
  • CSS Box Model = Margin > Border > Padding > Element/Content
  • The * selector indicates all of the elements on the page will be affected
  • To reference a class in CSS, use a period ( . ) plus the class’ name
  • Box-shadow = added a shadow to the element
  • Colors in CSS:
    • Semantic references e.g. blue or pink
    • Hexadecimals e.g. #0000FF or #FFC0CB
    • RGB values e.g. rgb(0,0,255) or rgb(255,192,203)

Git

  • git command = must precede every Git statement
  • git status = To check what branch we are on. If there are no branches, as at the beginning of the project, we should be on main. It should say “On branch main”
  • checkout command = is to move the working branch to a new branch.
  • The -b = creates a new branch.
  • git checkout -b feature/[name of new branch] = add a new branch to work on
  • git commit -m “[message]” = the git commit command with a commit message that describes what the commit contains. Always add a message when committing.
  • git pull origin main =
    • The pull command to receive a branch's modifications into the local environment.
    • The origin keyword indicates the source of the pull will be in the GitHub repo.
    • The main keyword indicates the branch.
  • git push origin [branch name] = a new command push to push the changes we have locally to our remote GitHub branch.
    • When using the push command, you can create a new remote branch or add new changes to an existing branch.

JavaScript

  • Variable: a named container that allows us to store data in our code
  • Var = how you declare a variable, followed by an equal sign =, then by the name you want to name the new variable enclosed in double-quotes “ ” then a ;
    • E.g. var colour = “red”;
  • Console.log = a method that outputs a message to the web console by adding an argument
    • An argument is what goes in the brackets
    • Console refers to a test environment that developers use to check out their code
    • .log() = will output whatever we add inside the parentheses to the console
    • Method = a set of instructions that can be executed by a computer
  • How to link the JavaScript file to the HTML file:
    • At the bottom of the body tag, above the ending tag, add the script tag
    • In the opening tag, add the attribute src
    • As the value for src, add the location of the JavaScript file
  • How to access the console in a webpage:
    • Right-click > Inspect > Click the "Console" tab
  • If statements - it will check to see if the condition that we provide is truthy (true)
    • If the condition isn’t true or has a value that isn’t truthy (false), then the computer will ignore whatever is in between the curly brackets and move on.
  • Syntax: if (condition) { code }
  • If else if statements - we can have more than one condition. If the first one is false, we can add another condition
  • Syntax: if (condition) { code } else if (condition) { code }
  • Difference between == and ===:
    • == is if they equal the same, whilst === strictly compares
    • var a = 10, b = ‘10’ -> a == b is true but a === b is false
  • Array = a single variable that is used to hold a group of data
    • Typically used to hold data that is related in some way
    • Arrays are not limited to contain only a single type of data
    • Declare an array using: var name_of_array = [item1, item2, …];
    • E.g. var schools = [“Belvedere Academy”, “St Margret’s”, “Liverpool College”];
  • To access an item in an array: Indexing
    • Index is the number placement the item is in the array
    • Indexing starts with 0 and not 1
    • Syntax: name_of_array[number]
    • E.g. schools[1] would equal “St Margret’s”
  • For loops = a loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Function = function is simply a set of instructions that tells the computer how to perform a certain task.
  • Syntax: function function_name() { code }
